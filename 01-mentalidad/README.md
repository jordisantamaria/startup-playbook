# 01 - Mentalidad

## Eres dev. Esa es tu ventaja injusta.

La mayoria de founders gastan meses buscando CTO, negociando equity, pagando freelancers. Tu construyes en un finde lo que ellos tardan 3 meses.

## La trampa del builder

Tu zona de confort es construir. El peligro es:
- Pasar 3 meses construyendo algo que nadie quiere
- Añadir features en vez de lanzar
- Perfeccionar en vez de validar

**La regla**: si llevas mas de un finde sin lanzar, estas procrastinando disfrazado de trabajar.

## Principios

### 1. Ship > Plan
No necesitas business plan, ni pitch deck, ni estudio de mercado. Necesitas un producto que alguien use y pague.

### 2. Un finde, una idea
El coste de probar una idea es un fin de semana + $300. Si no funciona, no has perdido nada. Si funciona, tienes un negocio.

### 3. No te encariñes
70 de los 71 productos de Pieter Levels fracasaron. El numero 71 fue Nomad List. La diferencia entre el y los demas: el lanzo los 70 anteriores en vez de quedarse pensando.

### 4. Cobra desde el dia 1
Si nadie paga, no hay negocio. No ofrezcas gratis "para conseguir usuarios". El unico usuario que importa es el que paga.

### 5. Solo hasta que no puedas solo
No busques co-founder porque "es lo que se hace". Busca co-founder cuando tengas revenue y necesites alguien que haga algo que tu no puedes (ventas enterprise, por ejemplo).

## Errores comunes del dev founder

| Error | Realidad |
|-------|----------|
| "Primero construyo bien, luego vendo" | Construye feo, lanza rapido, mejora con feedback real |
| "Necesito un co-founder de negocio" | No, necesitas 1 cliente que pague |
| "El marketing no es para mi" | Build in public ES tu marketing. Documentar lo que construyes |
| "Mi idea es demasiado simple" | TypingMind es un wrapper de ChatGPT y factura $50k/mes |
| "Ya existe competencia" | Mejor. Significa que hay mercado. Hazlo mejor o mas simple |
| "Necesito inversion para empezar" | Necesitas $300 (Supastarter) + un finde |

## Tu rutina

```
Viernes noche: elige la idea, haz fork de Supastarter
Sabado: construye el core feature con Claude Code
Domingo manana: polish, landing page, conecta Stripe
Domingo tarde: prepara assets para Product Hunt
Lunes: lanza
Martes-viernes: mide, itera si hay traccion, o empieza a pensar la siguiente
```

## Lectura recomendada

- **[MAKE](https://readmake.com)** — Pieter Levels. 2-3 horas. El playbook.
- **[The Mom Test](https://momtestbook.com)** — Rob Fitzpatrick. Para cuando quieras hablar con usuarios antes de construir (approach alternativo, valido para B2B).
