# 01 - Ideacion y Definicion de Producto

## El problema primero, la solucion despues

El error mas comun es enamorarte de una solucion sin validar que el problema exista. Tu trabajo en esta fase es **encontrar un problema que duela lo suficiente como para que alguien pague por resolverlo**.

## Framework: De problema a producto

### Paso 1: Identificar problemas

Fuentes de problemas reales:
- **Tu propia experiencia**: Que te frustra en tu trabajo/vida diaria?
- **Comunidades online**: Reddit, Twitter, foros de nicho. Busca quejas recurrentes
- **Tendencias macro**: Regulaciones nuevas, cambios tecnologicos, shifts demograficos
- **Trabajos anteriores**: Ineficiencias que viste y nadie resolvia

**Ejercicio**: Lista 10 problemas que hayas experimentado en los ultimos 30 dias. Para cada uno:
- Quien mas tiene este problema?
- Cuanto pagarian por resolverlo?
- Con que frecuencia ocurre?

### Paso 2: Evaluar el problema

Usa este scorecard (1-5 en cada criterio):

| Criterio | Pregunta | Score |
|----------|----------|-------|
| **Dolor** | Es un "nice-to-have" o un "must-have"? | /5 |
| **Frecuencia** | Ocurre diario, semanal, mensual? | /5 |
| **Mercado** | Cuanta gente tiene este problema? | /5 |
| **Disposicion a pagar** | Ya pagan por soluciones parciales? | /5 |
| **Accesibilidad** | Puedes llegar a estos usuarios? | /5 |
| **Competencia** | Las soluciones actuales son buenas? | /5 |

**Umbral**: Si score < 18, busca otro problema. Si >= 22, tienes algo interesante.

### Paso 3: Definir tu propuesta de valor

Usa esta plantilla:

```
Para [SEGMENTO DE CLIENTE]
Que tienen el problema de [PROBLEMA]
Nuestro producto es [CATEGORIA]
Que [BENEFICIO PRINCIPAL]
A diferencia de [ALTERNATIVA ACTUAL]
Nosotros [DIFERENCIADOR CLAVE]
```

**Ejemplo**:
```
Para freelancers que facturan a clientes internacionales
Que tienen el problema de perder tiempo extrayendo datos de facturas manualmente
Nuestro producto es un extractor automatico de facturas
Que convierte cualquier factura a datos estructurados en segundos
A diferencia de hacerlo a mano o usar software de contabilidad complejo
Nosotros usamos AI vision para extraer datos sin configuracion
```

### Paso 4: Lean Canvas (1 pagina)

Completa esto en 20 minutos, no mas:

| Seccion | Contenido |
|---------|-----------|
| **Problema** | Top 3 problemas |
| **Segmento** | A quien resuelves el problema |
| **Propuesta de valor** | Mensaje claro y simple |
| **Solucion** | Top 3 features |
| **Canales** | Como llegas al cliente |
| **Revenue** | Como ganas dinero |
| **Costes** | Principales costes |
| **Metricas clave** | Que mides para saber si funciona |
| **Ventaja injusta** | Que tienes que otros no pueden copiar facilmente |

## Herramientas utiles

- **Lean Canvas**: [leanstack.com](https://leanstack.com)
- **Jobs to be Done framework**: Para entender motivaciones profundas
- **Design Thinking**: Para empatizar con el usuario

## Errores comunes

1. **Construir sin hablar con nadie**: "Yo se lo que la gente quiere" - No, no lo sabes
2. **Mercado demasiado amplio**: "Mi app es para todo el mundo" = no es para nadie
3. **Enamorarte de la tecnologia**: La tecnologia es el medio, no el fin
4. **Copiar sin diferenciar**: Si haces lo mismo que el lider, por que cambiarian?
5. **Ignorar el timing**: Un buen producto en mal momento fracasa igual

## Prompt AI: Brainstorming de ideas

```
Actua como un consultor de startups experimentado.
Quiero explorar oportunidades en el sector de [TU SECTOR].

Mi experiencia es en: [TU BACKGROUND]
Tengo acceso a: [TUS RECURSOS/SKILLS]
Mi presupuesto inicial es: [PRESUPUESTO]

Dame 10 ideas de productos/servicios que:
1. Resuelvan un problema real y frecuente
2. Se puedan validar en menos de 2 semanas
3. Tengan potencial de escalar
4. Aprovechen mi experiencia y recursos

Para cada idea incluye:
- El problema especifico
- Quien lo tiene (segmento)
- Como monetizarlo
- Que necesito para un MVP
- Competencia existente y diferenciador posible
```
