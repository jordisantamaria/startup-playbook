# 04 - Launch

## $0 en marketing. Lanza donde la gente ya esta.

No compres ads. No hagas SEO todavia. No crees contenido. Simplemente pon tu producto delante de gente que puede usarlo.

## Donde lanzar (en orden)

### 1. Product Hunt (el mas importante)

**Preparacion** (domingo tarde):
- Crea maker profile si no tienes
- Assets: logo, tagline (5-10 palabras), descripcion (3-4 parrafos), screenshots, GIF/video de 1 min
- Consigue 10-20 amigos/conocidos que upvoteen y comenten el dia del launch (no bots, personas reales)

**El dia**:
- Publica a las 00:01 PST (09:01 hora España)
- Responde TODOS los comentarios
- Comparte el link en Twitter, LinkedIn
- No pidas "upvote" — pide "check it out and let me know what you think"

**Resultados esperados**:
- Top 5 del dia: 500-2000 visitas
- Top 1: 2000-5000+ visitas
- La mayoria no convertiran. OK. Es branding + primeros early adopters

**Mejor dia**: martes o miercoles

### 2. Hacker News (Show HN)

**Formato**:
```
Show HN: [Nombre] – [descripcion corta del problema que resuelve]

Contexto: tenia este problema [PROBLEMA], probe [ALTERNATIVAS] y no me convencian.
Asi que lo construi yo este finde.

Stack: Next.js, Postgres, Stripe.

Link: [URL]

Feedback bienvenido!
```

**Tips**:
- Se tecnico y directo. Nada de marketing-speak
- Si eres dev y lo construiste tu, les gusta
- Responde todo, acepta criticas con humildad
- No pongas precio en el post (eso lo ven en la web)

### 3. Twitter/X

**Tu primer tweet**:
```
I built [PRODUCTO] this weekend.

Problem: [1 frase]
Solution: [1 frase]

[screenshot o GIF]

Try it: [link]
```

**Thread de build in public** (mas engagement):
```
Tweet 1: I just launched [PRODUCTO]. Here's how I built it in 48 hours 🧵

Tweet 2: The problem: [descripcion personal del dolor]

Tweet 3: Saturday: I forked [boilerplate], set up [DB], and built [core feature]

Tweet 4: Sunday: Landing page, Stripe, deploy. The hardest part was [X]

Tweet 5: Monday (today): launched on Product Hunt. [link]

Tweet 6: Results so far: [numeros honestos]

Tweet 7: What's next: [1-2 things]. Try it and let me know: [link]
```

### 4. Reddit

Busca el subreddit de tu nicho. No spamees — aporta valor:

```
"Hey, I had this problem with [X] and couldn't find a good solution,
so I built one. Would love your feedback: [link]"
```

Subreddits generales: r/SaaS, r/indiehackers, r/startups, r/SideProject

### 5. Indie Hackers

Post de "Launch":
- Cuenta el problema, el proceso, muestra numeros reales
- La comunidad valora la transparencia

### 6. DevHunt, BetaList, Uneed

Directorios de productos. Menos trafico que PH pero SEO backlinks y early adopters.

## Que NO hacer en el launch

- No compres ads (todavia no sabes si el producto funciona)
- No contactes prensa/periodistas (no les importas todavia)
- No hagas cold email masivo
- No crees 5 cuentas de redes sociales
- No gastes dinero en branding profesional

## Medir el launch

| Metrica | Buena senal | Mala senal |
|---------|-------------|------------|
| Signups primera semana | >50 | <10 |
| Pagos primera semana | >1 | 0 |
| Comentarios/feedback | La gente pide features | Silencio total |
| Retention (vuelven?) | Usan el producto la semana 2 | Prueban y desaparecen |

## Post-launch decision

```
¿Alguien ha pagado?
├── SI → Felicidades. Sigue iterando → ve a 05-Grow
├── Hay signups pero no pagos → Ajusta pricing/onboarding, dale 1 semana mas
└── Nada → Next idea. No es personal. Es data.
```
